# パスワードやユーザー変数などの機密情報の扱い方

## いつでもチェックしたい注意点
- 機密情報はgitに上げない。  
    機密情報のファイルがある場合は`.gitignore`に追記してgitで無視されるようにする。
- 平文でパスワードなどをファイル(compose.yamlなど)に書かない

## docker コンテナ使用時のコツ
- パスワードやAPIキーなどはcompose secretsを使用してファイルベースでコンテナに渡す  
    利点
    - 必要なコンテナのみに明示的に情報を渡せる(compose.yamlを確認すれば機密情報をどのコンテナに渡しているかすぐわかる)
    - アプリ側はファイルを読み込めばいいので扱いやすい
        ```readFileSync("/run/secrets/db_password", "utf8").trim()```など
    - 環境変数は一覧でチェックされることもあるので意図せずもれやすい  
        一方でファイルは明示的に指定しないと開かれない
